# Predição de Perda

O módulo de Predição de Perda é o primeiro serviço inteligente da plataforma. Ele utiliza um **classificador binário LightGBM** para prever a probabilidade de perda de carga para cada CTRC (embarque) no momento da emissão, permitindo intervenção proativa antes que as perdas ocorram.

## Como funciona

O pipeline de predição segue um fluxo direto:

<Steps>
<Step title="Dados chegam">
Quando novos dados de CTRC são carregados no data warehouse (via pipeline ETL da `base_455`), o data warehouse publica um evento no tópico Pub/Sub `data-warehouse-updates`.
</Step>
<Step title="Serviço é acionado">
O serviço de Predição de Perda (executando no Cloud Run) recebe o evento e identifica novos CTRCs que ainda não foram pontuados pela versão ativa do modelo.
</Step>
<Step title="Modelo pontua CTRCs">
O modelo LightGBM analisa 19 features por CTRC — incluindo características da rota, atributos da carga, padrões temporais e dados históricos — para produzir uma probabilidade de perda entre 0 e 1.
</Step>
<Step title="Níveis de risco são atribuídos">
Com base na probabilidade, cada CTRC recebe um nível de risco:

| Nível de Risco | Descrição |
|-----------|-------------|
| **ALTO** | Maior probabilidade prevista de perda |
| **MEDIO** | Risco moderado que requer atenção |
| **BAIXO** | Baixa probabilidade de perda |

</Step>
<Step title="Resultados aparecem na plataforma">
As predições são escritas na tabela `ctrc_loss_predictions` no Cloud SQL e ficam imediatamente disponíveis na visualização em tabela e no coletor da plataforma.
</Step>
</Steps>

<Callout type="info">
O modelo usa uma **estratégia de exclusão por LEFT JOIN** — ele pontua apenas CTRCs que ainda não possuem uma predição para a versão ativa do modelo. Isso garante eficiência e evita pontuações duplicadas.
</Callout>

## Entendendo as predições

### Níveis de risco

Cada predição atribui um dos três níveis de risco com base na probabilidade de saída do modelo:

| Nível | Cor do Badge | O que significa |
|-------|------------|---------------|
| **ALTO** | Vermelho | Este CTRC tem o maior risco previsto de perda de carga. Requer atenção imediata e verificação. |
| **MEDIO** | Amarelo/Âmbar | Risco moderado. Deve ser monitorado e verificado quando possível. |
| **BAIXO** | Verde | Baixo risco. O manuseio de rotina é suficiente. |

### Probabilidade

A probabilidade bruta é um valor entre 0 e 1, exibido como percentual (ex.: 87,3%). Uma barra visual acompanha o número para dar uma noção rápida do risco relativo. A cor da barra corresponde ao badge do nível de risco.

### Confiança

O modelo também gera uma **pontuação de confiança** que reflete o quão certo o modelo está sobre sua predição. Maior confiança significa que as features do modelo se alinham fortemente com os padrões aprendidos durante o treinamento.

## Features do modelo

O modelo LightGBM utiliza **19 features** para fazer predições. Essas incluem (mas não se limitam a):

- Características da rota (origem, destino, distância)
- Atributos da carga (peso, valor, tipo de mercadoria)
- Padrões temporais (dia da semana, mês, época do ano)
- Estatísticas por unidade (taxas históricas de perda nas unidades de origem/destino)
- Metadados do embarque (número de volumes, valor do frete)

<Callout type="info">
O modelo é treinado com dados históricos de CTRCs onde o resultado (perda ou não perda) é conhecido. Ele aprende padrões que distinguem embarques de alto risco dos de baixo risco e então aplica esses padrões para pontuar novos embarques em tempo real.
</Callout>

## Versionamento do modelo

O serviço de Predição de Perda suporta **versionamento de modelos** sem necessidade de reimplantação do serviço:

- Os artefatos do modelo são armazenados no Google Cloud Storage (GCS)
- Um arquivo `manifest.json` no GCS define a versão ativa do modelo
- Quando um novo modelo é treinado e enviado, atualizar o manifest direciona todas as novas predições para a nova versão
- Predições históricas mantêm seu `model_version` original para rastreabilidade

Isso significa que a equipe de ciência de dados pode iterar nos modelos, comparar desempenho e implementar melhorias sem nenhuma mudança de infraestrutura.

## Fluxo operacional

O fluxo de trabalho recomendado para usar a Predição de Perda combina a Visualização em Tabela e o Coletor:

<Steps>
<Step title="Monitore a tabela">
Gestores de operações revisam a tabela de Predição de Perda diariamente, ordenada por probabilidade (maior primeiro), filtrada por níveis de risco ALTO + MEDIO.
</Step>
<Step title="Identifique embarques de alto risco">
Procure por concentrações de CTRCs de alto risco em unidades específicas. Isso pode indicar problemas sistemáticos nessas unidades.
</Step>
<Step title="Despache verificação em campo">
Compartilhe o código da unidade com operadores de campo, que abrem o Coletor em seus dispositivos móveis para verificar cargas de alto risco com evidência fotográfica.
</Step>
<Step title="Exporte para relatórios">
Use a funcionalidade de exportação CSV para baixar os dados de predição para análise offline, relatórios gerenciais ou integração com outros sistemas.
</Step>
</Steps>
