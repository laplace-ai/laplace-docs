# Database Browser

The Database Browser provides direct access to the data tables that power the Laplace Platform. It is useful for data validation, investigating specific shipments, and understanding the underlying information.

Navigate to **Database** in the sidebar.

## Interface layout

The Database Browser uses a **two-panel layout**:

### Left panel — Table list

A sidebar (within the main content area) lists all available tables. Click a table name to load its data into the main grid.

Available tables:

| Table | Description |
|-------|-------------|
| **CTRCs** | Shipment records — the core operational data |
| **Loss Predictions** | Prediction outputs including probability, risk level, and confidence |
| **Units** | Facility data with codes, names, locations, and geographic coordinates |
| **Network Links** | Route connections between facilities with vehicle type, classification, and pricing |

### Right panel — Data grid

The main area displays a scrollable table with the selected table's data.

## Available tables

### CTRCs

The CTRCs table contains shipment data. This is the primary operational dataset.

| Column | Description |
|--------|-------------|
| **CTRC** | Shipment document number |
| **Created** | Date the CTRC was created |
| **Current Location** | Current location of the cargo |
| **Origin Unit** | Emitting unit (origin facility) |
| **Destination Unit** | Receiving unit (destination facility) |
| **Sender** | Sender client name |
| **Recipient** | Recipient client name |
| **Origin City** | Origin city |
| **Delivery City** | Delivery city |
| **Delivery State** | Delivery state (e.g., SP, PR, RJ) |
| **Freight Value** | Freight value in BRL |
| **Cargo Value** | Cargo value in BRL |
| **Weight (kg)** | Actual cargo weight in kilograms |
| **Volumes** | Number of cargo volumes |
| **Goods** | Type of merchandise |

### Loss Predictions

The predictions table contains the scoring results for CTRCs.

| Column | Description |
|--------|-------------|
| **CTRC** | The scored shipment number |
| **Probability** | Loss probability (0% to 100%) |
| **Risk Level** | ALTO, MEDIO, or BAIXO |
| **Confidence** | How certain the model is about the prediction |
| **Prediction** | Whether the model predicts loss or no loss |
| **Model Version** | Which version of the model produced this prediction |

### Units

The units table contains data for all facilities in the logistics network.

| Column | Description |
|--------|-------------|
| **Code** | Unique facility code (e.g., "ATI SPA") |
| **Type** | Facility type (hub, branch, distribution center, etc.) |
| **Name** | Facility name |
| **City** | City location |
| **State** | State abbreviation |
| **Active** | Whether the unit is currently active |
| **Coordinates** | Geographic latitude and longitude (used for map placement) |

### Network Links

The network links table shows route connections between facilities.

| Column | Description |
|--------|-------------|
| **Origin** | Origin facility code |
| **Destination** | Destination facility code |
| **Vehicle Type** | Type of vehicle on this route |
| **Classification** | Route classification |
| **Price (BRL)** | Route pricing in BRL |

## Searching

Each table view includes a search bar. Type any value and the table instantly filters to rows containing that text in any column.

The search is:

- **Case-insensitive**: "sao paulo" matches "Sao Paulo"
- **Substring matching**: "SPA" matches "ATI SPA" and "SPA001"
- **Cross-column**: Searches across all columns simultaneously

The result count updates in real time, showing the number of filtered rows vs. total rows (e.g., "23 of 150 rows").

## Data loading

The Database Browser shows **up to 100 rows** per table. This is designed as a quick-look tool rather than a full data export interface.

When you switch between tables:

- The search query is cleared
- A loading indicator appears while data is fetched
- The new table's data replaces the previous view

<Callout type="info">
The Database Browser shows a snapshot of the first 100 rows from each table. For full data access, use the CSV export feature in the Loss Prediction table.
</Callout>

## Use cases

The Database Browser is particularly useful for:

| Use Case | How |
|----------|-----|
| **Data validation** | Verify that data was loaded correctly by spot-checking rows |
| **Investigation** | Look up a specific CTRC to see its details and prediction |
| **Unit lookup** | Find a facility's location, type, or status |
| **Route analysis** | Browse network links to understand route connections and pricing |
