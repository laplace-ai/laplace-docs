# Settings

The Settings page is the administrative hub of the Laplace Platform. It provides tabs for managing users, access profiles, and your own account. What you see depends on your role — regular users see only their account tab, while administrators have access to management features.

Navigate to **Settings** from the sidebar (gear icon at the bottom) or from the user dropdown menu.

## Tabs by role

| Tab | Visible to | Purpose |
|-----|-----------|---------|
| **Users** | Administrators | Manage users within your organization |
| **Access Profiles** | Administrators | Create and manage access profile templates |
| **My Account** | All users | View profile and change password |

Regular users land on the **My Account** tab by default. Administrators land on the **Users** tab.

## Users tab

The Users tab lets administrators manage all users in your organization.

{/* TODO: Add screenshot of the Users tab */}

### User list

The tab displays a table of all users with:

- **Name**: The user's full name
- **Email**: Their login email address
- **Role**: Badge showing Administrator or User
- **Access Profile**: The assigned profile (or "No profile" if unassigned)
- **Status**: Active, Inactive, or Pending (awaiting invite acceptance)
- **Actions**: A three-dot menu with context-specific actions

A user count badge shows the total number of users (e.g., "4 users").

### Inviting a new user

<Steps>
<Step title="Click Invite User">
Click the **Invite User** button (with a + icon) at the top of the tab.
</Step>
<Step title="Fill out the form">
A dialog appears with fields for:

- **Name**: The new user's full name
- **Email**: Their email address (must be unique)
- **Role**: Select Administrator or User from the dropdown
- **Access Profile**: Select a profile template that defines feature access
- **Facility Restrictions**: Optionally restrict the user to specific facilities (leave empty for all facilities)

</Step>
<Step title="Send the invite">
Click **Send Invite**. The system creates the user account, generates an invite token, and sends an email with the activation link.
</Step>
</Steps>

A toast notification confirms the invite was sent (e.g., "Invite sent to maria@company.com").

### Editing a user

Click the three-dot menu on a user's row and select the appropriate action:

| Action | Description |
|--------|-------------|
| **Edit User** | Opens a dialog to change the user's name, role, or access profile |
| **Facility Access** | Opens a dialog to manage which facilities the user can access |
| **Resend Invite** | Sends a new invite email (for users with `pending` status) |
| **Deactivate** | Disables the user's account (they can no longer log in) |
| **Activate** | Re-enables a previously deactivated user |

### Facility access management

The **Facility Access** dialog shows a checkbox list of all units in your network. You can:

- **Select All**: Grant access to every facility
- **Clear All**: Remove all facility restrictions (equivalent to full access)
- **Individual selection**: Check specific facilities to restrict the user to only those locations

When no facilities are selected, the user has access to all facilities (no restrictions). This is the default behavior.

The dialog shows a summary: "All facilities (no restrictions)" or "X facility(ies) selected".

<Callout type="info">
Facility access restrictions affect what data the user sees in all features — the map shows only their permitted units, the loss prediction table shows only CTRCs from their facilities, and so on.
</Callout>

## Access Profiles tab

Access profiles are templates that define which platform features a group of users can access. They provide a convenient way to manage access for different roles in your organization without configuring each user individually.

{/* TODO: Add screenshot of the Access Profiles tab */}

### Profile list

The tab shows all access profiles with:

- **Name**: The profile name (e.g., "Analyst", "Collector", "Manager")
- **Description**: An optional description of the profile's purpose
- **Features**: Badges showing the assigned features

A profile count badge shows the total (e.g., "3 profiles").

### Creating an access profile

<Steps>
<Step title="Click Create Profile">
Click the button at the top of the tab.
</Step>
<Step title="Fill out the form">
A dialog appears with:

- **Name**: A descriptive name (e.g., "Analyst", "Collector", "Manager")
- **Description**: Optional explanation of the profile's purpose
- **Feature Access**: A checklist of all available features. Select the features this profile should have access to.

</Step>
<Step title="Save">
Click **Create**. The profile is created and can now be assigned to users.
</Step>
</Steps>

### Editing an access profile

Click on a profile in the list to open the edit dialog. You can change the name, description, and feature selection. Changes apply immediately to all users assigned this profile.

### Deleting an access profile

Click the delete option in the profile's context menu. A confirmation dialog asks "Are you sure you want to delete this access profile?" before proceeding.

<Callout type="warning">
Deleting an access profile affects all users currently assigned to it. Those users will lose their feature assignments and may see a reduced sidebar until a new profile is assigned.
</Callout>

### Available features

The feature access checklist includes all available features in the platform:

| Feature | Description |
|---------|-------------|
| Digital Twin | Interactive logistics network map |
| Digital Model | Network modeling |
| Units | Unit/facility management |
| Loss Prediction | Cargo loss probability predictions |
| Loss Prediction Table | Table view of predictions |
| Loss Prediction Collector | Mobile verification interface |
| Loss Prediction Dashboard | Analytics dashboard |
| Loss Prediction Models | Model management |
| Control Tower | Real-time operations monitoring (coming soon) |
| Load Consolidation | Shipment consolidation (coming soon) |
| Database Browser | Raw data exploration |

The summary shows how many features are selected (e.g., "5 feature(s) selected").

## My Account tab

The My Account tab is available to all users. It shows your profile information and lets you change your password.

### Account information

A read-only section displays:

| Field | Description |
|-------|-------------|
| **Name** | Your full name |
| **Email** | Your login email address |
| **Role** | Your role (Administrator or User) |
| **Organization** | The organization you belong to |

### Change password

Below the account information, a password change form lets you update your password:

<Steps>
<Step title="Enter current password">
Type your current password to verify your identity.
</Step>
<Step title="Enter new password">
Type your new password (minimum 8 characters).
</Step>
<Step title="Confirm new password">
Re-type the new password for confirmation.
</Step>
<Step title="Submit">
Click **Update Password**. A toast notification confirms the change.
</Step>
</Steps>

**Validation rules:**

- New password must be at least 8 characters
- New password and confirmation must match
- Current password must be correct

<Callout type="tip">
You can change your password at any time. If you forget your password, ask your administrator to resend your invite link, which will let you set a new one.
</Callout>

All Settings labels, buttons, and messages are fully translated in both English (EN-US) and Portuguese (PT-BR). The language used in Settings follows your global language preference, which you can change via the language toggle on the login page or in the platform header.
